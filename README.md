# 코딩테스트란?

- 3대장: C++, Java, Python
- 알고리즘 문제풀이(Problem Solving): PS

- 표준 입출력, 표준 STL 사용, 일관적이고 공정한 방식이다.
- 수많은 응시자들을 순위 매기기 좋다.
- 문제 해결 능력 + 코딩 능력을 단시간에 측정 가능

## 코딩테스트를 잘하려면?

- 알고리즘 문제 푸는 과정에서 요구되는 요소들

  - 독해력: 문제를 빠르고 정확하게 파악
  - 배경지식: 필수 자료구조, 알고리즘 지식들
  - 문제해결력: 솔루션, 아이디어
  - 구현력: 코딩, 구현 문제가 있을 수도 있음(아이디어보다 구현능력을 보는 문제)
  - 검증, 디버깅: 틀렸을 경우 틀린 곳 or 반례 찾기

- 필수 자료구조, 알고리즘을 먼저 훑고, 문제해결력을 트레이닝 한 뒤, 구현력을 키우자

## 코딩테스트 출제 경향

### 삼성

- 3시간 2문제
- 사용가능 언어: C, C++, Java, Python
- 시뮬레이션, 구현, DFS/BFS를 주료 출제

### LINE

- 3시간 5~6 문제
- 사용 가능언어: C++, Java, Javscript, Python, Kotlin, Swift
- 구현, 탐색, 시뮬레이션, 자료구조 등

### Kakao

- API를 이용하는 형태
- 5시간 7문제
- 사용 가능언어: C++, Java, Javascript, Python, Kotlin, Swift
- 구현, 문자열, 탐색, DP, DFS/BFS, 그래프 등 범위가 넓음
- 가장 어렵다
- 프로그래머스에서 풀 수 있음

## 채점 기준

- 부분 점수 유무
- 문제마다 배점 동일 여부
- 제출 시도 횟수 제한 유무
- 채점 결과 공개 유무
  ...

### 일반적으로

- 푼 문제 수가 많을 수록 순위가 높음
- 같은 문제 수라면 더 적은 시도 횟수/ 더 빨리 맞췄을 수록 순위가 높음
- 절반 이상 풀었을 경우 합격권인 경우가 많음
  - 삼성은 2문제중 1문제 이상, 카카오는 7문제 중 4문제 정도
  - 문제 난이도와 응시자들 실력에 따라 달라질 수 있음
- 문제마다 존재하는 제한 조건
  - 시간 제한: 시간 복잡도
  - 메모리 제한: 공간 복잡도
  - 일반적으로 시간과 공간은 trade-off 관계이다.

### 문제 해결 시작

- 변수, 함수, 배열, 반복문, 조건문, 문자열 등을 주로 사용하며, 상속, 포인터 등은 거의 사용하지 않음
- 사용하는 문법은 한정적이다.
- 입출력 함수와 빠른 입출력 함수: n = int(input()), a, b = map(int, input().split())
- 입력 데이터가 매우 많으면, 빠른 입출력을 사용하자

```python
# bad
for _ in range(100000):
  n = int(input())
  print(n)

# good
for _ in range(100000):
  n = int(sys.stdin.readline()) # 속도가 빠름
  print(n)
```
